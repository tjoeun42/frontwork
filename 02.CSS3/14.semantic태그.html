semantic 태그

의미가 있는 태그

  • html4버전에서는 div태그의 id등을 통해 내용을 구분
  • 스크린리더는 영역별로 접근 불가
  • html5버전에서는 태그의 이름으로 태그의 내용을 유추
  • 스크린리더는 영역별로 접근 가능
  • 웹 접근성 향상을 위하 태그

사용 태그

    <header>
        헤더 부분 : 맨 상위에 위치하고, 회사 로고나 기타 헤더에 넣을 것들을 배치
    </header>

    <nav>
        nav : 다른페이지로 이동할 수 있는 페이지 네비게이션 정보. 위치가 가장 자유롭다
    </nav>

    <section>
        section : 연관된 컨텐츠 그룹핑(본문)
    </section>

    <article>
        article : 실제 내용을
            - section으로 여러개의 article을 그룹핑 가능
            - article본문이 section으로 나뉠 수 있음
    </article>

    <aside>
        aside : 본문 외의 부가정보
            - 날씨, 광고, 네비게이션 등
    </aside>

    <footer>
        footer : 웹 페이지 하단부분 또는 article의 하단에 위치
            - 회사정보, 저작권 정보, 네이게이션 등
    </footer>
